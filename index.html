Projects

What I Have Done, Am Doing, Will Do...


Austin Web Development Boot Camp Application
This application was developed within a MEAN stack framework. This was a solo project specifically to showcase some end of class chops. It was a great learning experience - I enjoyed the creative control of a streamlined style. I found most of my time was spent wrangling the back-end and aligning the database HTTP get/pull/post methods. Express is a great framework for Node set up. Using a non-relational database in this and previous projects was eye opening as most of my experience was with SQL databases. It is clear how much the non-relational databases (MongoDB) have grown in use as the amount of web based chat and data transfer escalates.


GearTracker - the application to monitor all or your monitors!
The GearTracker app was a group endeavor to emphasize skills with MongoDB and Node. We created an existing inventory list of band gear to select from which initiated the individual gear list that was tracked for the logged-in user. The user could checkout equipment that was unloaded into a venue and check it back in once the performance was complete and the equipment was loaded into the van. We went through a few iterations on how best to align the collection as an array for the user in the MongoDB document and then be able to separate the items out to track individually. We utilized our diagrams of data flow repeatedly - eventually we just took a picture of it for reference. You may have guessed that one our team members was in a band!


BOBOPOLY - Austin Board Games and More
The Bobopoly aplication was built to demonstrate our experience with Ruby on Rails. We decided to build an informational platform for Austin based board game enthusiasts based on our classmate Bob (a gaming wizard!). We displayed basic game information and brought in specific venue information for playing board games. We tracked user login so we could untilize a discussion forum for that specific community. As a team we divided our tasks and tracked them via Pivotal Tracker. Along with side-by-side coding we had individual responsibilities. I personally worked through the user-authentication and comment section pieces and presented the code to the team. We broke out styling, session logic, and tabbed views as separate tasks for the team to bring to the table. We were also able to use RSPEC to incorporate testing while developing the code.


The Classic Game of TicTacToe
This was one of our first projects to bring in javascript logic with CSS and HTML files. I was able to have fun with this project and bring in some CSS animation with this project for a little zing effect when the page loads. Be warned this was a first attempt at a project so perfection was not the goal - this is where we learned the concept of MVP! Take it for a spin, grab a friend - click on the game image and you two can play TicTacToe.
